# Plan: Databricks Upload Helper Script

## Overview

A standalone interactive CLI script (`scripts/upload_to_databricks.py`) that:
1. Connects to Azure Databricks using a PAT stored in `.env`
2. Prompts user to pick which local SQLite model to upload
3. Prompts for Unity Catalog destination (`catalog.schema.table`)
4. Checks user privileges before transferring data
5. Warns if target table exists; asks overwrite / append / cancel
6. Batch-inserts all rows from the local SQLite table into a Delta table
7. Reports progress and final count

---

## User Decisions

| Question | Answer |
|----------|--------|
| SQL Warehouse HTTP path source | `.env` — `DATABRICKS_HTTP_PATH` |
| Namespace level | Unity Catalog — `catalog.schema.table` |
| Columns to upload | All columns including `data_json` blob |

### Privilege Check Detail

Four checks are done sequentially; each failure is clearly reported:

| Check | SQL | Required Privilege |
|-------|-----|--------------------|
| **Auth** | `SELECT CURRENT_USER()` | Valid PAT |
| **Catalog visible** | `SHOW CATALOGS LIKE '<cat>'` | `USE CATALOG` |
| **Schema accessible** | `SHOW SCHEMAS IN <cat> LIKE '<schema>'` | `USE SCHEMA` (or schema doesn't exist yet) |
| **Table readable** (if table exists) | `SELECT COUNT(*) FROM <cat>.<schema>.<table>` | `SELECT` |
| **Write** | Attempted during CREATE/INSERT; caught + reported clearly | `CREATE TABLE`, `MODIFY` |

Notes:
- `SHOW GRANTS` is intentionally skipped — it requires metastore admin rights.
- If catalog/schema doesn't exist yet, the check passes with a note:
  `"Schema not found — will attempt to create it during upload."`
- Actual write-privilege errors are surfaced at transfer time with a clear
  message explaining which privilege is missing (parsed from exception text).

---

### INSERT Strategy

- Fetch all rows from SQLite as a `list[dict]`
- Escape string values: replace `'` with `''` (standard SQL escaping)
- `None` values → SQL `NULL`
- Batch size: **500 rows per INSERT statement** (well within Databricks SQL limits)
- Format: `INSERT INTO cat.schema.table VALUES (...), (...), ...`
- Progress: Print `Uploaded N / Total ...` every batch

### Edge Cases Handled

| Case | Handling |
|------|---------|
| Local SQLite table is empty | Warn and exit gracefully |
| Schema doesn't exist yet | Auto-create with `CREATE SCHEMA IF NOT EXISTS` |
| Table exists + overwrite | `DROP TABLE` then `CREATE TABLE ... USING DELTA` |
| Table exists + append | `CREATE TABLE IF NOT EXISTS` (DDL no-op), then INSERT |
| Invalid destination name | Regex validate before sending to Databricks |
| Connection timeout | Surface clear error with troubleshooting tips |
| Permission denied | Parse exception message, show required privilege |
| Partial failure mid-batch | Log which batch failed, continue remaining batches, report at end |
| `data_json` contains single quotes | Escaped via `str.replace("'", "''")` |
| `data_json` is None | Written as SQL `NULL` |

### 1. Color — Breaking Changes

**Old assumed structure** (what the code was built against):
```python
{
  "id": ...,
  "headerNumber": ...,   # ← used by upsert_color()
  "headerName": ...,     # ← used by upsert_color()
  ...
  "colors": [...],       # ← used by colors_page, push_color()
```

**Actual current structure**:
```python
{
  "id": ...,
  "colorPaletteNumber": "LB-S12027-00003",   # ← RENAMED field
  "colorPaletteName": "LOFT SS27 Color...",   # ← RENAMED field
  "folder": { "id": ..., "name": ... },
  "createdBy": { "id": ..., "name": ... },    # ← now an object
  "createdAt": ...,
  "modifiedBy": { "id": ..., "name": ... },   # ← now an object
  "modifiedAt": ...,
  "headerData": {
    "fields": [...],                          # unchanged
    "colors": {                               # ← MOVED & RESTRUCTURED
      "ASEurl": "",
      "colors": [                             # ← now nested inside headerData.colors
        {
          "_id": ...,
          "color_id": ...,
          "hex": "e0aac8",
          "color_number": "14-5412 TCX",      # ← key renamed (was colorNumber)
          "color_name": "Frostbite",          # ← key renamed (was colorName)
          "rgb_r", "rgb_g", "rgb_b", ...      # ← full CIE/RGB/CMYK data

          "Schema": { ... },
          ...
        }
      ]
    }
  },
  "isDeleted": false                          # ← new field

```

**Impact**:
- `upsert_color()` uses `record.get("headerNumber")` / `record.get("headerName")` → **always empty now** — all synced color records will have NULL `header_number` / `header_name`
- `colors_page.py` reads `data.get("colors", [])` → **returns empty list**; colors are at `data["headerData"]["colors"]["colors"]`
- `push_color()` sends `colors = data.get("colors") or []` → **sends empty array**; loses all color data on push
- Color swatch keys changed: `colorNumber` → `color_number`, `colorName` → `color_name`

## Files to Change

### `app/db.py`

**Changes:**

1. **`_DDL`** — Add new tables, add migration-safe ALTER for `directory.modified_at`:

   ```sql
   CREATE TABLE IF NOT EXISTS images ( ... );
   CREATE TABLE IF NOT EXISTS blocks ( ... );
   ```

2. **`init_schema()`** — After `executescript(_DDL)`, run a silent `ALTER TABLE directory ADD COLUMN modified_at TEXT` (wrapped in try/except, since SQLite raises OperationalError if column already exists).

3. **Helper `_extract_active_from_fields(record)`** — New helper that reads `headerData.fields` for the `active` field:
   ```python
   def _extract_active_from_fields(record: dict) -> int:
       fields = record.get("headerData", {}).get("fields", [])
       for f in fields:
           if f.get("id") == "active":
               val = f.get("value", "")
               return 1 if str(val).lower() in ("yes", "true", "1") else 0
       return 0
   ```

4. **`upsert_style()`** — Change active extraction: use `_extract_active_from_fields(record)` instead of `record.get("active", "false")`.

8. **`upsert_directory_record()`** — Three fixes:

```sql
    header_number TEXT,
    header_name   TEXT,
```

# Plan: Incorporate Latest BeProduct Data Structure Changes

## Executive Summary

Sampled live API data for all requested models. Several structural changes have occurred since
the local store was built. The plan below documents every divergence found and the exact changes
needed in each file.

---

## What Was Sampled

All entities were fetched from the live API via the SDK:

| Entity | SDK call |
|--------|----------|
| Style | `client.style.attributes_list()` |
| Material | `client.material.attributes_list()` |
| Color | `client.color.attributes_list()` |
| Image | `client.image.attributes_list()` (new) |
| Block | `client.block.attributes_list()` (new) |
| Directory | `client.directory.directory_list()` |
| Users | `client.user.user_list()` (new) |

## Findings: Data Structure Changes by Model

### 1. Color — Breaking Changes

**Old assumed structure** (what the code was built against):
```python
{
  "id": ...,
  "headerNumber": ...,   # ← used by upsert_color()
  "headerName": ...,     # ← used by upsert_color()
  ...
  "colors": [...],       # ← used by colors_page, push_color()
}
```

**Actual current structure**:
```python
{
  "id": ...,
  "colorPaletteNumber": "LB-S12027-00003",   # ← RENAMED field
  "colorPaletteName": "LOFT SS27 Color...",   # ← RENAMED field
  "folder": { "id": ..., "name": ... },
  "createdBy": { "id": ..., "name": ... },    # ← now an object
  "createdAt": ...,
  "modifiedBy": { "id": ..., "name": ... },   # ← now an object
  "modifiedAt": ...,
  "headerData": {
    "fields": [...],                          # unchanged
    "colors": {                               # ← MOVED & RESTRUCTURED
      "ASEurl": "",
      "colors": [                             # ← now nested inside headerData.colors
        {
          "_id": ...,
          "color_id": ...,
          "hex": "e0aac8",
          "color_number": "14-5412 TCX",      # ← key renamed (was colorNumber)
          "color_name": "Frostbite",          # ← key renamed (was colorName)
          "rgb_r", "rgb_g", "rgb_b", ...      # ← full CIE/RGB/CMYK data
          "palette_name": ...,
          "Schema": { ... },
          ...
        }
      ]
    }
  },
  "isDeleted": false                          # ← new field
}
```

**Impact**:
- `upsert_color()` uses `record.get("headerNumber")` / `record.get("headerName")` → **always empty now** — all synced color records will have NULL `header_number` / `header_name`
- `colors_page.py` reads `data.get("colors", [])` → **returns empty list**; colors are at `data["headerData"]["colors"]["colors"]`
- `push_color()` sends `colors = data.get("colors") or []` → **sends empty array**; loses all color data on push
- Color swatch keys changed: `colorNumber` → `color_number`, `colorName` → `color_name`

### 2. Style — New Fields Added (non-breaking, but worth capturing)

Top-level structure now includes:
```python
{
  "id": ...,
  "headerNumber": ...,   # unchanged
  "headerName": ...,     # unchanged
  "folder": { ... },
  "headerData": {
    "fields": [...],
    "frontImage": { "preview": "...", "origin": "..." },   # NEW
    "sideImage":  { "preview": "...", "origin": "..." },   # NEW
    "backImage":  { "preview": "...", "origin": "..." },   # NEW
    "availableArtboards": [ { "artboarIndex": 0, "imageUrl": "..." }, ... ],  # NEW
  },
  "createdBy": { "id": ..., "name": ... },    # now an object (was a string)
  "modifiedBy": { "id": ..., "name": ... },   # now an object (was a string)
  "createdAt": ...,
  "modifiedAt": ...,
  "colorways": [
    {
      ...,
      "hideColorway": false,        # NEW
      "imageHeaderId": null,        # NEW
      "colorSourceId": null,        # NEW
      "secondaryColorNumber": "",   # NEW
      "secondaryColorName": "",     # NEW
    }
  ],
  "sizeRange": [
    {
      ...,
      "hideSize": false,    # NEW
      "sizeIndex": 1,       # NEW
      "fields": {},         # NEW
    }
  ],
  "sizeClasses": [           # NEW top-level array
    {
      "id": ...,
      "name": "Boys Standard - Alpha",
      "isDefault": true,
      "sizeRange": [ ... ]  # same shape as sizeRange
    }
  ],
  "planIds": null,                    # NEW
  "isDeleted": false,                 # NEW
  "copiedOrCarriedOverFrom": null,    # NEW
  "active": null,                     # top-level active is NULL (was absent before)
}
```

**Key issue**: `active` is NOT at the top level (returns `null`). The `active` value is
inside `headerData.fields` as `id="active"`, `type="TrueFalse"`. The current upsert code
does `record.get("active", "false")` which always evaluates to 0 for all styles. The "Active"
column in the UI list always shows ❌ incorrectly.

**Fix**: Extract `active` from `headerData.fields` using a helper.

### 4. Directory — Schema Changes (breaking for push)

```python
# ACTUAL current response
{
  "id": ...,
  "directoryId": ...,
  "name": ...,
  "address": ...,
  "country": ...,
  "state": ...,
  "zip": ...,
  "city": ...,
  "phone": ...,
  "partnerType": "SUPPLIER",   # NEW value (old code only expected VENDOR/FACTORY/AGENT/RETAILER/OTHER)
  "website": ...,
  "createdAt": ...,             # NEW
  "modifiedAt": ...,            # NEW
  "createdBy": { "id": ..., "name": ... },    # NEW
  "modifiedBy": { "id": ..., "name": ... },   # NEW
  # MISSING: fax, active, contacts
}
```

**Removed fields** (no longer returned by API):
- `fax` — never returned; `push_directory()` sends it unnecessarily
- `active` — never returned; `upsert_directory_record()` extracts it and sets it to 0 always; `push_directory()` sends it
- `contacts` — not in directory list/get response; contacts are accessed separately via `directory_contact_list(id)` but no contacts existed in test data

**New fields**: `createdAt`, `modifiedAt`, `createdBy`, `modifiedBy`

**Impact**:
- Directory `active` column in DB is always 0 (API no longer returns it)
- Directory list page shows "❌" for Active on all rows
- `push_directory()` sends fields (`fax`, `active`) that may be rejected by API
- `directory_page.py` tries to display `fax` field which is always empty

## Files to Change

### `app/db.py`

**Changes:**

1. **`_DDL`** — Add new tables, add migration-safe ALTER for `directory.modified_at`:
   ```sql
   CREATE TABLE IF NOT EXISTS images ( ... );
   CREATE TABLE IF NOT EXISTS blocks ( ... );
   CREATE TABLE IF NOT EXISTS users ( ... );
   ```

2. **`init_schema()`** — After `executescript(_DDL)`, run a silent `ALTER TABLE directory ADD COLUMN modified_at TEXT` (wrapped in try/except, since SQLite raises OperationalError if column already exists).

3. **Helper `_extract_active_from_fields(record)`** — New helper that reads `headerData.fields` for the `active` field:
   ```python
   def _extract_active_from_fields(record: dict) -> int:
       fields = record.get("headerData", {}).get("fields", [])
       for f in fields:
           if f.get("id") == "active":
               val = f.get("value", "")
               return 1 if str(val).lower() in ("yes", "true", "1") else 0
       return 0
   ```

4. **`upsert_style()`** — Change active extraction: use `_extract_active_from_fields(record)` instead of `record.get("active", "false")`.

5. **`upsert_material()`** — Same as style active fix.

6. **`upsert_color()`** — Three fixes:
   - Extract `header_number` from `record.get("colorPaletteNumber")` (was `headerNumber`)
   - Extract `header_name` from `record.get("colorPaletteName")` (was `headerName`)
   - Use `_extract_active_from_fields(record)` for active

7. **`update_color_local()`** — Fix `header_name` update to use `updated_json.get("colorPaletteName")`.

8. **`upsert_directory_record()`** — Three fixes:
   - Remove `active` extraction (default to 0 since API no longer returns it)
   - Add `modified_at` parameter in INSERT/UPDATE using `record.get("modifiedAt")`
   - Keep the existing columns but don't fail on missing `active`

9. **Add Image CRUD** (`upsert_image`, `get_images`, `get_image`, `update_image_local`, `mark_image_clean`)

10. **Add Block CRUD** (`upsert_block`, `get_blocks`, `get_block`, `update_block_local`, `mark_block_clean`)

11. **Add User CRUD** (`upsert_user`, `get_users`, `get_user`) — read-only (no push, no dirty)

12. **`get_row_counts()`** — Extend to include images, blocks, users tables in counts.

### `app/push.py`

**Changes:**

1. **`push_color()`** — Fix colors extraction. The `colors` list now lives at
   `headerData.colors.colors` (a `dict` wrapper, not a bare list). Each color object
   has keys like `color_number`, `color_name`, `hex`, `color_id`, `color_source_id`, etc.
   The SDK's `attributes_update(colors=...)` accepts the list of these color objects as-is:
   ```python
   # OLD (broken — returns empty list):
   colors = data.get("colors") or []
   # NEW:
   colors_wrapper = data.get("headerData", {}).get("colors") or {}
   colors = colors_wrapper.get("colors") or []
   ```

2. **`push_directory()`** — Remove `fax`, `active`, `contacts` from the fields dict sent
   to the API; they are no longer returned by the API and may cause push errors:
   ```python
   # Remove these from the fields dict:
   # "fax": ..., "active": ..., "contacts": ...
   ```

## SQLite Schema Additions

```sql
CREATE TABLE IF NOT EXISTS images (
    id            TEXT PRIMARY KEY,
    folder_id     TEXT,
    folder_name   TEXT,
    header_number TEXT,
    header_name   TEXT,
    active        INTEGER,
    created_at    TEXT,
    modified_at   TEXT,
    synced_at     TEXT,
    is_dirty      INTEGER DEFAULT 0,
    data_json     TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_images_folder    ON images(folder_id);
CREATE INDEX IF NOT EXISTS idx_images_modified  ON images(modified_at);
CREATE INDEX IF NOT EXISTS idx_images_dirty     ON images(is_dirty);

CREATE TABLE IF NOT EXISTS blocks (
    id            TEXT PRIMARY KEY,
    folder_id     TEXT,
    folder_name   TEXT,
    header_number TEXT,
    header_name   TEXT,
    active        INTEGER,
    created_at    TEXT,
    modified_at   TEXT,
    synced_at     TEXT,
    is_dirty      INTEGER DEFAULT 0,
    data_json     TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_blocks_folder    ON blocks(folder_id);
CREATE INDEX IF NOT EXISTS idx_blocks_modified  ON blocks(modified_at);
CREATE INDEX IF NOT EXISTS idx_blocks_dirty     ON blocks(is_dirty);

CREATE TABLE IF NOT EXISTS users (
    id            TEXT PRIMARY KEY,
    email         TEXT,
    username      TEXT,
    first_name    TEXT,
    last_name     TEXT,
    title         TEXT,
    account_type  TEXT,
    role          TEXT,
    registered_on TEXT,
    active        INTEGER,
    synced_at     TEXT,
    data_json     TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_users_active ON users(active);
```

Migration for existing `directory` table:
```python
# In init_schema(), after executescript(_DDL):
try:
    with get_conn() as conn:
        conn.execute("ALTER TABLE directory ADD COLUMN modified_at TEXT")
except Exception:
    pass  # Column already exists
```

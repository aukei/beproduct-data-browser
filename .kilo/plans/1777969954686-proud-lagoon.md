# Databricks ETL Pipeline: BeProduct Master Data Sync

## Overview

Create a bidirectional Databricks ETL pipeline that:
1. **Downloads** each BeProduct master (Styles, Materials, Colors, Images, Blocks, Directory, Users, DataTables) directly from BeProduct API to independent Databricks Delta tables
2. **Pushes back** curated/updated data from Databricks to BeProduct with merge/conflict resolution

This enables:
- Real-time analytics on BeProduct PLM data via Databricks
- Data enrichment workflows in Databricks
- Write-back capability to sync corrections/updates to BeProduct

---

## Architecture Decisions

### 1. Direct API → Delta (Not SQLite → Delta)
- **Rationale**: Eliminates need for local sync; data flows directly from BeProduct API to Databricks
- **Advantage**: Scales to large catalogs without local storage bottleneck
- **Trades**: Must handle pagination & error recovery in Databricks notebooks (not Python scripts)

### 2. Independent Master Jobs
- **One job per master** (Styles, Materials, Colors, etc.)
- **Rationale**: Allows parallel execution; failures in one don't block others
- **Scheduling**: Can be staggered or run on different cadences

### 3. Two Job Categories
- **Inbound**: BeProduct API → Databricks Delta (read-only on BeProduct side, append-only or merge in Databricks)
- **Outbound**: Databricks Delta → BeProduct API (pull curated/modified data, merge with remote state)

#### 3.1 Template Structure (all inbound notebooks follow this pattern)
```python
# ============================================================================
# Databricks Notebook: Sync [Master] from BeProduct to Delta
# ============================================================================
# 
# Purpose: Download all [Master] records from BeProduct API, transform,
#          and write to Delta table. Supports full + incremental modes.
#
# Schedule: [e.g., hourly, nightly]
# Parameters:
#   - incremental_mode (bool): if True, only fetch records modified since last sync
#   - target_catalog (str): Unity Catalog name
#   - target_schema (str): Schema name
#   - batch_size (int): rows per commit (default 1000)

import logging
from datetime import datetime, timezone

# Databricks imports
from pyspark.sql import SparkSession
from pyspark.sql.types import StructType, StructField, StringType, LongType, TimestampType

# Local imports (must be in same workspace)
from beproduct_api_utils import BeProductClient, normalize_style_row, setup_spark_session

# ============================================================================
# PARAMETERS
# ============================================================================

dbutils.widgets.dropdown("incremental_mode", "false", ["true", "false"])
dbutils.widgets.text("target_catalog", "main")
dbutils.widgets.text("target_schema", "beproduct")
dbutils.widgets.text("batch_size", "1000")

incremental_mode = dbutils.widgets.get("incremental_mode") == "true"
target_catalog = dbutils.widgets.get("target_catalog")
target_schema = dbutils.widgets.get("target_schema")
batch_size = int(dbutils.widgets.get("batch_size"))

table_name = f"{target_catalog}.{target_schema}.bp_styles"

# ============================================================================
# SETUP
# ============================================================================

logger = logging.getLogger(__name__)
logging.basicConfig(level=logging.INFO)

spark = setup_spark_session(target_catalog, target_schema)

# Retrieve BeProduct credentials from Secrets
client_id = dbutils.secrets.get(scope="beproduct", key="client_id")
client_secret = dbutils.secrets.get(scope="beproduct", key="client_secret")
refresh_token = dbutils.secrets.get(scope="beproduct", key="refresh_token")
company_domain = dbutils.secrets.get(scope="beproduct", key="company_domain")

client = BeProductClient(client_id, client_secret, refresh_token, company_domain)

# ============================================================================
# FETCH DATA
# ============================================================================

sync_start_time = datetime.now(timezone.utc)
sync_batch_id = f"beproduct_sync_{sync_start_time.timestamp()}"

logger.info(f"Starting [Master] sync. Batch ID: {sync_batch_id}")
logger.info(f"Mode: {'Incremental' if incremental_mode else 'Full'}")

# Determine incremental filter
incremental_filter = None
if incremental_mode:
    # Query last sync time from _delta_log or a metadata table
    try:
        last_sync_meta = spark.sql(f"""
            SELECT MAX(synced_at) as last_sync_at 
            FROM {table_name}
        """).collect()
        incremental_filter = last_sync_meta[0]["last_sync_at"] if last_sync_meta[0]["last_sync_at"] else None
        logger.info(f"Incremental filter: synced_at > {incremental_filter}")
    except Exception as e:
        logger.warning(f"Could not read last sync time, falling back to full: {e}")
        incremental_mode = False

# Fetch from API
rows = []
try:
    for api_row in client.fetch_styles(incremental_filter=incremental_filter):
        normalized_row = normalize_style_row(api_row)
        normalized_row["synced_at"] = sync_start_time.isoformat()
        normalized_row["_databricks_modified_at"] = sync_start_time
        normalized_row["_databricks_modified_by"] = "databricks_job"
        normalized_row["_sync_batch_id"] = sync_batch_id
        rows.append(normalized_row)
        
        # Batch insert
        if len(rows) >= batch_size:
            df = spark.createDataFrame(rows)
            # MERGE if table exists, INSERT otherwise
            _merge_or_insert(df, table_name, "id")
            rows = []
    
    # Final batch
    if rows:
        df = spark.createDataFrame(rows)
        _merge_or_insert(df, table_name, "id")
        rows = []
    
    logger.info(f"✅ Sync complete. Total rows: {len(rows)}")

except Exception as e:
    logger.error(f"❌ Sync failed: {e}", exc_info=True)
    raise

# ============================================================================
# HELPER: MERGE OR INSERT
# ============================================================================

def _merge_or_insert(df, full_table_name, key_col):
    """Insert new rows or update existing by key."""
    try:
        df.write.format("delta") \
            .mode("overwrite") \
            .option("mergeSchema", "true") \
            .saveAsTable(full_table_name)
    except Exception:
        # Fallback to append if first run
        df.write.format("delta") \
            .mode("append") \
            .option("mergeSchema", "true") \
            .saveAsTable(full_table_name)

# ============================================================================
# LOG RESULTS
# ============================================================================

result_df = spark.sql(f"SELECT COUNT(*) as total_records FROM {table_name}")
total = result_df.collect()[0]["total_records"]
logger.info(f"✅ Final table state: {total} records in {table_name}")

print(f"✅ Sync complete")
```

#### 4.2 Template Outbound Notebook
```python
# ============================================================================
# Databricks Notebook: Push Modified [Master] to BeProduct
# ============================================================================
# 
# Purpose: Read modified records from Delta table, merge with latest
#          from BeProduct API, and push updates back to BeProduct.
#
# Schedule: [e.g., hourly]
# Parameters:
#   - dry_run (bool): if True, log changes but don't push
#   - conflict_strategy (str): "local_wins" | "remote_wins" | "manual_review"

import logging
from datetime import datetime, timezone

from pyspark.sql import SparkSession
from beproduct_push_utils import BeProductPusher, detect_conflict, merge_records

# ============================================================================
# PARAMETERS
# ============================================================================

dbutils.widgets.dropdown("dry_run", "true", ["true", "false"])
dbutils.widgets.dropdown("conflict_strategy", "manual_review", ["local_wins", "remote_wins", "manual_review"])
dbutils.widgets.text("target_catalog", "main")
dbutils.widgets.text("target_schema", "beproduct")

dry_run = dbutils.widgets.get("dry_run") == "true"
conflict_strategy = dbutils.widgets.get("conflict_strategy")
target_catalog = dbutils.widgets.get("target_catalog")
target_schema = dbutils.widgets.get("target_schema")

# ============================================================================
# SETUP
# ============================================================================

spark = SparkSession.getActiveSession()
logger = logging.getLogger(__name__)

# Credentials
client_id = dbutils.secrets.get(scope="beproduct", key="client_id")
client_secret = dbutils.secrets.get(scope="beproduct", key="client_secret")
refresh_token = dbutils.secrets.get(scope="beproduct", key="refresh_token")
company_domain = dbutils.secrets.get(scope="beproduct", key="company_domain")

pusher = BeProductPusher(client_id, client_secret, refresh_token, company_domain)

# ============================================================================
# FETCH MODIFIED RECORDS
# ============================================================================

table_name = f"{target_catalog}.{target_schema}.bp_styles"

# Get records marked for push (e.g., a column `_push_pending = 1`)
modified_df = spark.sql(f"""
    SELECT * FROM {table_name}
    WHERE _push_pending = 1
""")

modified_records = modified_df.collect()

logger.info(f"Found {len(modified_records)} records to push")

if not modified_records:
    logger.info("✅ No records to push")
else:
    # ========================================================================
    # PUSH WITH MERGE
    # ========================================================================
    
    push_results = []
    
    for record in modified_records:
        record_id = record["id"]
        
        try:
            # Fetch latest from BeProduct API
            remote = pusher.client.fetch_style_by_id(record_id)
            
            # Detect conflict
            if detect_conflict(record.asDict(), remote):
                logger.warning(f"Conflict detected for {record_id}")
                
                if conflict_strategy == "manual_review":
                    push_results.append({
                        "id": record_id,
                        "status": "conflict",
                        "message": "Conflict detected; requires manual review"
                    })
                    continue
                elif conflict_strategy == "remote_wins":
                    logger.info(f"Skipping {record_id} (remote_wins strategy)")
                    push_results.append({
                        "id": record_id,
                        "status": "skipped",
                        "message": "Remote version is newer"
                    })
                    continue
            
            # Merge if safe
            merged = merge_records(record.asDict(), remote, strategy=conflict_strategy)
            
            # Push to BeProduct
            if not dry_run:
                success, msg = pusher.push_style(merged, overwrite_conflicts=(conflict_strategy == "local_wins"))
                
                if success:
                    # Mark as synced (remove _push_pending flag)
                    spark.sql(f"""
                        UPDATE {table_name}
                        SET _push_pending = 0, _push_synced_at = CURRENT_TIMESTAMP()
                        WHERE id = '{record_id}'
                    """)
                    push_results.append({
                        "id": record_id,
                        "status": "success",
                        "message": msg
                    })
                else:
                    push_results.append({
                        "id": record_id,
                        "status": "failed",
                        "message": msg
                    })
            else:
                logger.info(f"[DRY RUN] Would push {record_id}")
                push_results.append({
                    "id": record_id,
                    "status": "dry_run",
                    "message": "Dry run — not pushed"
                })
        
        except Exception as e:
            logger.error(f"Error pushing {record_id}: {e}", exc_info=True)
            push_results.append({
                "id": record_id,
                "status": "error",
                "message": str(e)
            })
    
    # Log results
    result_df = spark.createDataFrame(push_results)
    display(result_df)
    
    success_count = len([r for r in push_results if r["status"] == "success"])
    logger.info(f"✅ Push complete: {success_count}/{len(push_results)} succeeded")

print("✅ Done")
```

## Key Implementation Challenges & Solutions

### Challenge 1: OAuth Token Refresh in Databricks
**Problem**: Databricks notebooks don't have persistent state between runs.

**Solution**: 
- Use `client_secret + refresh_token` to request new access token on each job run
- Implement token caching with TTL (8 hours) in `BeProductClient` class
- Handle 401 responses by refreshing and retrying

### Challenge 2: Pagination & Rate Limits
**Problem**: Large catalogs (>10K styles) require pagination; must not exceed rate limits.

**Solution**:
- Implement exponential backoff in `paginate_api_call()` on 429 responses
- Use `pageSize=1000` (max per API docs)
- Log rate limit headers on each request
- Consider staggering inbound jobs by time-of-day

### Challenge 3: Incremental Sync Filtering
**Problem**: BeProduct API's `FolderModifiedAt` filter may not match Delta table's `modified_at` exactly.

**Solution**:
- On first run, use `modified_at` from Delta's `_delta_log`
- Store last successful sync timestamp in a metadata table (e.g., `bp_sync_metadata`)
- On incremental runs, query metadata table for filter cutoff
- If metadata query fails, fall back to full sync with warning

### Challenge 4: Merge Conflicts
**Problem**: Data edited in Databricks AND in BeProduct concurrently.

**Solution**:
- Compare `modified_at` timestamps: whoever edited last "wins" (configurable strategy)
- Log conflicts to audit table for manual review
- Provide `conflict_strategy` parameter: "local_wins", "remote_wins", "manual_review"
- For manual_review, mark row as `_push_pending = 1` and skip until manually approved

### Challenge 5: Ordering & Dependencies
**Problem**: Directory data may be referenced by Styles/Materials; should sync Directory first.

**Solution**:
- Define sync order: Directory → Users → DataTables → Styles → Materials → Colors → Images → Blocks
- But since each job is independent, use `depends_on` in job definitions if needed
- OR use a multi-task job that runs sequentially

## Implementation Plan (Detailed)

### Inbound Jobs Schedule (Every 6 Hours)
```
00:00 → 01_sync_styles, 02_sync_materials (parallel)
      → 03_sync_colors, 04_sync_images (parallel)
      → 05_sync_blocks, 06_sync_directory (parallel)
      → 07_sync_users
      → 08_sync_data_tables (last)

06:00 → (repeat)
12:00 → (repeat)
18:00 → (repeat)
```
**Rationale**: Stagger by 30 min offsets to avoid thundering herd on API

### Outbound Jobs (On-Demand Only)
- No scheduled runs
- Triggered manually via Databricks Jobs UI or API
- All use `conflict_strategy = "local_wins"` by default
- All outbound jobs log to `main.beproduct.bp_audit_log`

### Audit Log Schema
```sql
CREATE TABLE main.beproduct.bp_audit_log (
    audit_id STRING,                      -- UUID
    timestamp TIMESTAMP,                   -- when the push occurred
    job_id STRING,                         -- Databricks job ID
    run_id STRING,                         -- Databricks job run ID
    master_type STRING,                   -- styles, materials, colors, etc.
    record_id STRING,                     -- BeProduct record UUID
    action STRING,                        -- INSERT, UPDATE, CONFLICT_SKIPPED, ERROR
    databricks_modified_at TIMESTAMP,     -- local version timestamp
    beproduct_modified_at TIMESTAMP,      -- remote version timestamp
    error_message STRING,                 -- if action = ERROR
    databricks_user STRING,               -- user who triggered the push job
    _databricks_modified_at TIMESTAMP,    -- system metadata
    _databricks_modified_by STRING
) USING DELTA
```

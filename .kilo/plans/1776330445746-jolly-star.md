# Plan: Bulk Push All Models to Databricks

## Goal

Update `scripts/upload_to_databricks.py` to push all 6 models — styles, materials, colors, images, blocks, directory — to Databricks in a single run. The user is asked for catalog, schema, and a table prefix. One Delta table is created per model, named `{prefix}_{model}`. The full API JSON is stored in the existing `data_json` text field.

---

## Changes Required

**File:** `scripts/upload_to_databricks.py` (only file to modify)

---

## Detailed Implementation Plan

### 1. Replace `MODELS` list with `BULK_MODELS`

Remove the `users` entry. The 6 models to push are:

```python
BULK_MODELS: list[tuple[str, str]] = [
    ("Styles",    "styles"),
    ("Materials", "materials"),
    ("Colors",    "colors"),
    ("Images",    "images"),
    ("Blocks",    "blocks"),
    ("Directory", "directory"),
]
```

Keep `MODELS` is no longer needed (or rename it). Keep `_ENTITY_COLUMNS` DDL dict intact — it already defines columns + `data_json STRING` for all 6 models.

---

### 2. Remove `_select_model()` function

No longer needed — all 6 models are always uploaded.

---

### 3. Replace `_get_destination()` with `_get_destination_bulk()`

```python
def _get_destination_bulk() -> tuple[str, str, str]:
    """Prompt for catalog, schema, and table prefix. Returns (catalog, schema, prefix)."""
    print()
    print(BOLD("Destination (Unity Catalog):"))
    catalog = _validate_identifier(_prompt("  Catalog", "main"), "Catalog")
    schema  = _validate_identifier(_prompt("  Schema"), "Schema")
    prefix  = _validate_identifier(_prompt("  Table prefix (tables will be prefix_styles, prefix_materials, …)"), "Table prefix")
    return catalog, schema, prefix
```

Table names are then computed as `f"{prefix}_{entity}"` for each entity.

---

### 4. Update `_check_privileges()` — check all 6 tables at once

Add a helper `_check_all_tables(cursor, catalog, schema, prefix, entities)` that:
- Checks catalog accessibility once
- Checks schema existence once  
- For each of the 6 entities, checks if `{prefix}_{entity}` already exists and its row count
- Returns a summary dict:
  ```python
  {
    "catalog_ok": bool,
    "schema_ok": bool,
    "tables": {
      "styles":    {"exists": bool, "row_count": int, "full_name": str},
      "materials": ...,
      ...
    }
  }
  ```

---

### 5. Ask once for overwrite/append/cancel (only if any tables already exist)

After the privilege check:
- If any of the 6 target tables already exist, display them and ask once:  
  `Action? [O]verwrite / [A]ppend / [C]ancel`
- If none exist, proceed directly to create mode.
- The chosen mode applies to **all** tables.

---

### 6. New `main()` flow

```
1. Load config
2. Print banner
3. Connect to Databricks + verify identity
4. Show local row counts for all 6 models
5. Prompt: catalog, schema, table prefix
6. Check privileges for all 6 target tables
7. If catalog not accessible → exit
8. If any tables exist → ask once for overwrite/append/cancel
9. Create schema if it doesn't exist (once)
10. Loop over each of the 6 models:
    a. Compute table name: {prefix}_{entity}
    b. Fetch local SQLite data
    c. Drop + recreate / create-if-not-exists / just append based on mode
    d. Upload rows in batches of 500
    e. Print per-model result line
11. Print final summary table (model | rows uploaded | rows failed | duration)
```

## Key Design Decisions

| Decision | Choice |
|---|---|
| Models to upload | styles, materials, colors, images, blocks, directory (no users) |
| Table naming | `{prefix}_{entity}` e.g. `beproduct_styles` |
| Conflict mode | Ask once, apply to all 6 tables |
| Full JSON storage | Existing `data_json STRING` column — no schema change needed |
| Schema creation | CREATE SCHEMA IF NOT EXISTS — runs once before the loop |
| Batch size | 500 rows per INSERT (unchanged) |
